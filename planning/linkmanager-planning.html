Other info

  1. see if can be used Tree from list
  2. see if is feasible to add all fields from CategoryMap in db ( for add / move / delete category -> recalculate DirectChildrenIds, DirectChildrenCount, RecursiveChildrenIds, RecursiveChildrenCount, ArticlesCount)
    Conclusion :: to keep these fields updated in database, it adds a lot of complexity, especially when when moving or deleting a category. For example, after a category is moved, all recursive children count and ids needs to be recalculated for the source category and all of its parents and also for the destination parent parent and all of its parents, and then update all these changes in database. Since the computation of this info is anyway made in the CategoryMap, there's no reason to add them to the database.
